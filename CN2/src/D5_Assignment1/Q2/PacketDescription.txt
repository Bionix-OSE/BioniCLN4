DATAGRAM FORMAT:
[4 bytes for sequence number][1 byte for data type][data bytes]

After the server had processed sequence number (which allows it to display the data in order), the 5th byte in the datagram is what going to let it determine the data type it's going to process next (and also doubles as the exit signal with "data type" 4).
